Meaning before implementation

Semantic validation for UAD 3.6 appraisal data

The UAD 3.6 Validation project ingests governed specification assets, projects XML instance documents into RDF, applies semantic validation, and produces measurable evidence about implementation quality.

The project is also the first reference implementation of Ontology Test-Driven Development (OTDD), an engineering methodology for semantic engineering.

Purpose

What this project is intended to do

Ingest governed assets

Load the UAD 3.6 schema, implementation guidance, controlled vocabularies, business rules, XLink relationships, and authoritative sample files.

Preserve meaning

Represent domain concepts and relationships as graph resources with stable identity, provenance, and traceability to their source artifacts.

Validate instances

Validate appraisal instance documents structurally and semantically using XML Schema, RDF, OWL, SHACL, and executable verification.

Produce measurements

Generate evidence about competency coverage, sample coverage, validation outcomes, missing semantics, and implementation quality.

Architecture

Semantic Engineering Architecture

UAD 3.6 Validation is organized around semantic engineering artifacts rather than traditional software layers. The architecture emphasizes meaning, traceability, provenance, validation, and measurable engineering outcomes.

Engineering Pipeline

  1. Authoritative specifications
  2. Semantic competency discovery
  3. Ontology development
  4. Metadata generation
  5. Example projection
  6. Validation execution
  7. Measurement generation
  8. Published reports

Repository Organization

Directory Purpose
app/ Application implementation and services.
ontologies/ OWL ontologies defining project semantics.
metadata/ Engineering metadata represented as RDF.
examples/ Representative valid and invalid appraisal instances.
features/ Executable behavior specifications.
reports/ Generated engineering outputs.
site/ Published engineering portal.

Primary Semantic Assets

  • UAD Core Ontology
  • URAR Meaning Model
  • SHACL Validation Shapes
  • OTDD Competency Metadata
  • OTDD Sample Metadata
  • Generated Measurements

Relationship to OTDD

The architecture follows the principles of Ontology Test-Driven Development by treating semantic engineering artifacts as first-class engineering deliverables. Every ontology, competency, example, validation, and measurement is intended to be traceable to an authoritative source.
